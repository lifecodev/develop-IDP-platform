# Построение высокодоступной DevOps-платформы для микросервисной архитектуры с использованием CI/CD, IDP, Kubernetes, Terraform и Vault
Выпускная квалификационная работа (ВКР) по направлению 09.03.01 «Информатика и вычислительная техника».

Специализация: DevOps / Platform Engineering
## 📌 О проекте
Данная выпускная квалификационная работа посвящена проектированию, реализации и автоматизации жизненного цикла микросервисных приложений в рамках отказоустойчивого (High-Availability) контура.

Основным результатом работы является создание комплексной экосистемы, объединяющей инфраструктуру как код (Terraform), оркестрацию (Kubernetes), централизованное динамическое управление секретами (HashiCorp Vault) и непрерывную интеграцию (GitLab CI/CD). Управление и взаимодействие команд с этой экосистемой инкапсулировано внутри единой внутренней платформы разработчика (Internal Developer Platform — IDP) на базе Spotify Backstage. Платформа позволяет абстрагировать разработчиков от сложности облачной инфраструктуры и ускорить Time-to-Market микросервисов.

## 🏗 Архитектура и Технологический стек
Проект реализует принципы декларативного описания инфраструктуры, концепцию безопасности Zero Trust (идентификация на основе ролей) и централизованную наблюдаемость.
- **Оркестрация микросервисов**: Высокодоступный кластер Kubernetes (k3s / HA-конфигурация).
- **Управление секретами**: HashiCorp Vault (динамическая генерация секретов, трансляция токенов, mTLS / TLS шифрование данных в покое и при передаче).
- **Infrastructure as Code (IaC)**: Terraform (модульное описание сетевой структуры, вычислительных нод и политик доступа) + Ansible (базовая конфигурация ОС и подготовка узлов кластера).
- **Слой IDP (Internal Developer Platform)**: Spotify Backstage (Каталог программного обеспечения, шаблонизатор Scaffolder для мгновенного старта новых микросервисов, плагины интеграции).
- **Служба непрерывной интеграции (CI/CD)**: Селф-хостед GitLab и изолированные GitLab Runners, интегрированные с API Kubernetes и Vault.
- **Наблюдаемость (Observability)**: Мониторинг здоровья микросервисов и контейнеров с использованием Prometheus, Grafana и Node Exporter.

## 🚀 Ключевой функционал платформы
1. Создание микросервиса в клик (Self-Service Scaffolding): Разработчик запускает шаблон в IDP, который автоматически генерирует репозиторий в GitLab, создаёт базовый пайплайн сборки, регистрирует сущность в каталоге и запрашивает необходимые политики в Vault.
2. Сквозная безопасность (Zero Trust Secrets): Микросервисы в Kubernetes динамически получают токены доступа к базам данных и API через Service Account JWT и интеграцию с HashiCorp Vault. Никаких хардкод-секретов в репозиториях.
3. Отказоустойчивость и Проксирование: Использование Nginx в качестве реверс-прокси с поддержкой trustProxy, корректной передачей заголовков (X-Forwarded-*) и защитой сессий (CSRF/Nonce) для безопасной работы OAuth 2.0 (GitLab Auth) в продакшене.
4. Мониторинг инфраструктуры: Кастомные Grafana-дашборды для отслеживания утилизации ресурсов кластера, запущенных подов и общего состояния хостов.

## 📁 Структура репозитория
```
├── terraform/             # Манифесты IaC для развертывания базовой инфраструктуры
│   ├── main.tf
│   ├── variables.tf
│   └── modules/           # Модули сети, compute-нод и k8s
├── ansible/               # Плейбуки для конфигурации нод кластера и окружения
├── kubernetes/            # Манифесты оркестрации и деплоя микросервисов
│   ├── core/              # Системные компоненты (Ingress, Cert-Manager)
│   └── apps/              # Конфигурации развертывания микросервисов
├── vault/                 # Конфигурация политик доступа и ролей HashiCorp Vault
├── backstage/             # Исходный код и продакшн-конфигурация ядра IDP
│   ├── app-config.yaml    # Базовые настройки
│   ├── app-config.production.yaml # Конфиг авторизации (GitLab OAuth, trustProxy)
│   └── Dockerfile         # Многоэтапная сборка платформы
├── monitoring/            # Конфигурационные файлы Prometheus, Grafana и экспортеров
├── docker-compose.yml     # Спецификация для локального тестирования/стейджинга платформы
└── README.md              # Текущий документ
```

📈 Результаты внедрения

- Время развертывания нового микросервиса (от создания репозитория до первого деплоя в K8s с мониторингом и секретами) сокращено до нескольких минут.\
- Когнитивная нагрузка на разработчиков снижена благодаря абстрагированию инфраструктурных манифестов в UI-шаблоны IDP.
- Устранена уязвимость хранения статических секретов благодаря динамическим токенам Vault.
